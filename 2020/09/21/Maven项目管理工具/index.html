安装

依赖传递

在没学Maven之前，普通的Java项目或者Web项目，依赖的导入是非常的不方便的。比如

原始java项目：创建个与src目录平级的libs/lib目录，导入所需jar包

原始web项目：在WEB-INF目录下创建个lib目录，导入所需jar包。

原来的缺点：不方便，不会自动帮我们处理依赖。例如：我们导入jackson的jar包,jackson 该jar包导入一个是不行的，应该其依赖的很多的其他jar包，我没猜错的话，是三个…

现在学了Maven后,我们只有要在pom.xml导入jackson的依赖坐标后，maven会自动帮我们优化处理相关的依赖。比如：

这种现象就是依赖传递

Maven项目生命周期

什么是生命周期？

生命周期就是从出生到死亡的一个过程

Maven项目的生命周期，简言之，就是几个命令

我自己认为比较重要的几个命令，有如下几个

  • clean 清理上次构建产物
  • compile 对项目进行编译
  • test 测试，test中的单元测试方法会自动执行
  • package 将项目打包
